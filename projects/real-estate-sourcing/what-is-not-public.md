# What Is Not Public — Real Estate Sourcing

This project is documented in a **confidentiality-conscious** way. The repository is designed to demonstrate capability and approach without exposing the sensitive assets that give the workflow its value or that could affect third parties.

## What Is Omitted

The following are intentionally **not** included in this repository:

- **Full n8n workflows** — no complete workflow exports or node-level configuration
- **Exact sourcing methods** — the specific sources and how opportunities are collected
- **Private databases** — any underlying data stores, schemas, or contents
- **Proprietary scoring rules** — the qualification criteria, weights, and thresholds
- **Prompts** — the exact LLM instructions used in the AI layer
- **Credentials** — API keys, tokens, and any authentication details
- **Client or lead data** — any personal, client, or counterparty information
- **Sensitive opportunity information** — any specific real estate opportunity details

## Why It Is Omitted

- **Business value.** The sourcing logic and qualification rules are the core know-how of the project; publishing them would give away what makes it useful.
- **Confidentiality and trust.** Client, lead, and opportunity data must remain private as a matter of professional responsibility.
- **Security.** Credentials and full workflow exports must never be exposed publicly.
- **Accuracy and context.** Out-of-context implementation details can be misleading without the surrounding business framing.

## What Is Shown Instead

To demonstrate the work transparently while protecting sensitive elements, the repository provides:

- A clear **business case** explaining the problem and the value of the approach
- A **high-level architecture** describing the staged pipeline conceptually
- The **stack and tools** used
- A **roadmap** showing how the project is intended to evolve
- An honest description of the **current status and limitations**

In other words, the repository shows the **reasoning, structure, and approach** — not the reproducible internals.

## Reassurance

This repository is deliberately structured to prove capability — the ability to understand a business problem, design a workflow, apply AI pragmatically, and document it clearly — **without exposing** proprietary logic, sensitive data, or anything that could compromise confidentiality or security. Sensitive details can be discussed in an appropriate, private setting where confidentiality is ensured.
