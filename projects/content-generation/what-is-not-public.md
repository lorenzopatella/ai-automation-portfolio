# What Is Not Public — Content Generation

This project is documented in a **confidentiality-conscious** way. The repository is designed to demonstrate capability and approach without exposing the sensitive assets that give the workflow its value or that could affect a brand or third parties.

## What Is Omitted

The following are intentionally **not** included in this repository:

- **Full n8n workflows** — no complete workflow exports or node-level configuration
- **Exact prompts** — the specific LLM and generation instructions used
- **Private brand assets** — logos, templates, style files, or other proprietary materials
- **Internal creative direction** — the underlying brand strategy, tone-of-voice rules, and creative guidelines
- **Credentials** — API keys, tokens, and any authentication details
- **API keys** — any keys or secrets for the connected services
- **Full production logic** — the complete end-to-end implementation details
- **Client or proprietary content** — any client work, deliverables, or owned content
- **Sensitive visual pipeline details** — the specific tools, settings, and steps used to produce visual content

## Why It Is Omitted

- **Creative value.** Prompts, creative direction, and production logic are the core know-how; publishing them would give away what makes the workflow effective.
- **Brand protection.** Brand assets and guidelines are proprietary and must remain private.
- **Confidentiality and trust.** Client and proprietary content must not be exposed as a matter of professional responsibility.
- **Security.** Credentials, API keys, and full workflow exports must never be published.
- **Accuracy and context.** Out-of-context implementation details can be misleading without the surrounding business framing.

## What Is Shown Instead

To demonstrate the work transparently while protecting sensitive elements, the repository provides:

- A clear **business case** explaining the problem and the value of the approach
- A **high-level architecture** describing the staged pipeline conceptually
- The **stack and tools** used
- A **roadmap** showing how the project is intended to evolve
- An honest description of the **current status and limitations**

In other words, the repository shows the **reasoning, structure, and approach** — not the reproducible internals.

## Reassurance

This repository is deliberately structured to prove capability — the ability to design a content production workflow, orchestrate tools, apply AI pragmatically, build for non-technical users, and document it clearly — **without exposing** prompts, brand assets, creative direction, or any client/proprietary content. Sensitive details can be discussed in an appropriate, private setting where confidentiality is ensured.
